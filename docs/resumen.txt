Resumen inicial proyecto 
1. Introducción 
El proyecto consiste en el desarrollo de una aplicación web destinada a mejorar y automatizar la gestión 
del préstamo de libros en el Instituto IES Arcipreste de Hita. Surge de la necesidad de sustituir los procesos 
manuales o basados en hojas de cálculo (Excel), que resultan poco eficientes y propensos a errores, por 
una solución centralizada, segura y accesible. 
La aplicación permitirá registrar y controlar los préstamos de libros concedidos a los alumnos becados, 
facilitando las tareas de administración, seguimiento y devolución de ejemplares, además de optimizar los 
tiempos de gestión y reducir costes operativos. 
2. Objetivos del Proyecto 
• Centralizar la gestión de libros y alumnos en una base de datos unificada. 
• Simplificar los procesos administrativos relacionados con la cesión, devolución y control del estado 
de los libros. 
• Ofrecer una interfaz web intuitiva accesible desde cualquier dispositivo con conexión a Internet. 
• Permitir la generación de listados y contratos de préstamo de forma automática. 
• Garantizar la seguridad y preservación de los datos, mediante copias de seguridad y control de 
accesos. 
3. Funcionalidad Esperada 
La aplicación se estructura en tres grandes áreas funcionales: 
A. Carga de datos 
• Importación de materias, cursos, libros y alumnado al inicio del curso escolar. 
• Posibilidad de limpiar y reiniciar la base de datos. 
• Gestión de errores durante la carga de información. 
B. Gestión de alumnos 
• Alta, modificación y baja de registros de alumnos. 
• Visualización del historial de préstamos de cada alumno. 
• Control de tramos de beca y asignaciones específicas. 
C. Gestión de préstamos 
• Asignación de libros a los alumnos y control de devoluciones. 
• Registro del estado de los ejemplares. 
• Generación e impresión de contratos de cesión. 
• Firma electrónica o validación administrativa.

D. Listados y consultas 
• Búsquedas avanzadas con filtros por curso, materia o estado. 
• Listados de alumnos, libros y préstamos. 
• Exportación de datos a formatos CSV o SQL para el curso siguiente. 
E. Administración y seguridad 
• Autenticación mediante roles (Administrador y Alumno). 
• Copia de seguridad y restauración de datos. 
• Logs de actividad y preservación de información sensible según la normativa de protección de 
datos. 
4. Tecnologías Utilizadas 
Lenguajes de Programación 
• PHP 8.2: lenguaje principal para el desarrollo del backend. 
• HTML5 y CSS3: estructura y diseño de las páginas web. 
• Bootstrap 5.3: framework CSS para lograr un diseño adaptable y profesional. 
Gestión de Bases de Datos 
• MySQL 8.2: motor de base de datos relacional utilizado para el almacenamiento de la información 
(libros, alumnos, préstamos). 
• SQL estructurado para creación, consultas y mantenimiento. 
Entorno de Desarrollo 
• Apache 2.4.54: servidor web para pruebas locales. 
• Visual Studio Code: entorno de desarrollo profesional para PHP. 
• Draw.io: herramientas para diseño de diagramas y prototipos. 
5. Estructura del Sistema 
El proyecto sigue una arquitectura modular, con separación entre capas: 
• Capa de presentación: interfaz web responsive basada en Bootstrap. 
• Capa de negocio: controladores PHP encargados de las operaciones de alta, baja y modificación. 
• Capa de datos: base de datos relacional MySQL con las tablas alumnos, libros, materias, cursos y 
alumnoscursoslibros. 
El sistema está diseñado para ser escalable y mantenible, permitiendo su posible integración futura con 
otras plataformas educativas. 




PARTE 1

Este proyecto consiste en desarrollar una aplicación web tipo Fantasy League, 
inspirada en Fantasy LaLiga, pero centrada en el mundo de la Fórmula 1. Los 
usuarios podrán crear equipos de pilotos y escuderías, participar en ligas 
privadas o públicas, y obtener puntos en función del rendimiento real de los 
pilotos en los Grandes Premios. El sistema incluirá gestión de usuarios y ligas, 
selección de pilotos con límite presupuestario, puntuaciones automáticas y 
clasificaciones. 
1.1 RECOPILACIÓN DE INFORMACIÓN 
Fuentes principales: webs oficiales de F1, APIs deportivas como Ergast API o F1 
Stats API, y ejemplos como F1 Fantasy o Biwenger.  
Contexto: creciente interés por los juegos de simulación y gestión deportiva 
online.  
Referencias técnicas: documentación de React, Node.js, Spring Boot y MySQL. 
1.2 OBJETIVOS, ALCANCE Y VIABILIDAD 
Objetivo general: crear una aplicación web gamificada de gestión de equipos de 
Fórmula 1. 
Objetivos específicos: diseñar un sistema de usuarios y ligas, implementar un 
motor de puntuación automática y crear una interfaz intuitiva y visual. 
Alcance: versión inicial centrada en la temporada actual, sin apuestas ni 
recompensas económicas. 
Viabilidad técnica: Frontend con JS y Bootstrap, backend con PHP, base de 
datos MySQL y despliegue en Docker. 
1.2.1 ANÁLISIS DAFO 
Fortalezas: idea atractiva, gran base de fans, escalable. 
Debilidades: dependencia de APIs externas, mantenimiento intensivo. 
Oportunidades: integración con redes sociales, creación de ligas locales o 
corporativas. 
Amenazas: derechos de datos oficiales, competencia de plataformas existentes.
